Especificaçăo Técnica – App de Análise de Pagamentos de Marketplace
Objetivo

Construir uma aplicaçăo local para carregar e analisar, de forma rápida e recorrente, os dados de pagamentos por ciclo de faturamento do marketplace, incluindo histórico de transaçőes, documentos contabilísticos (automáticos e manuais) e transferęncias bancárias (TRF). A app deve normalizar os dados, calcular KPIs chave (acumulados e por último ciclo) e disponibilizar um dashboard simples, além de endpoints para integraçăo.
1. Âmbito e KPIs
Fontes de dados a suportar:
* Histórico de transaçőes por ciclo (XLSX/CSV).
* Documentos contabilísticos (faturas/notas automáticas e manuais) e/ou “Resumo das transaçőes” (quando disponível).
* Transferęncias bancárias (TRF) em XLSX/CSV, layout variável.
KPIs obrigatórios (acumulado e “último ciclo”):
* Prazos médios de pagamento (apenas “Valor do pedido”): média, mínimo, máximo.
* Total de pedidos “Recebido”.
* Total de produtos vendidos.
* Reserva presa (saldo em aberto).
* Ciclo da constituiçăo da última reserva.
* Total de comissőes pagas (sem impostos) e Imposto sobre comissőes.
* Total de reembolsos incluindo impostos.
* Conciliaçăo: Net por ciclo vs TRF (+0/+7 dias) e diferença.
2. Arquitetura proposta
Stack: Python + DuckDB + FastAPI (API) + Pandas (ETL) + (Opcional) Streamlit/Plotly para UI.
Racional: zero dependęncias externas, rapidez em ficheiros grandes, portabilidade, e separaçăo clara entre ingestăo (ETL), armazenamento e exposiçăo de KPIs.

Tabelas principais:

5. KPIs – Definiçőes e Queries (DuckDB SQL)
5.1 Prazos médios (média/mín/máx) – apenas “Valor do pedido”:

WITH items AS (
  SELECT DATE_DIFF('day', "Data Criaçăo", "Data do ciclo de faturamento") AS dias
  FROM transactions WHERE LOWER(Tipo) = 'valor do pedido'
)
SELECT AVG(dias) AS prazo_medio_dias, MIN(dias) AS prazo_min_dias, MAX(dias) AS prazo_max_dias FROM items;

5.2 Último ciclo (mais recente por data de ciclo):

SELECT "Ciclo Pagamento"
FROM transactions
WHERE "Data do ciclo de faturamento" IS NOT NULL
ORDER BY "Data do ciclo de faturamento" DESC
LIMIT 1;

5.3 Comissőes (sem impostos) e Imposto sobre comissőes – acumulado e último ciclo:

-- Acumulado
SELECT
  SUM(CASE WHEN LOWER(Tipo) REGEXP '(taxa|taxas|taxa do vendedor|taxa de assinatura)' THEN Débito ELSE 0 END) AS comissoes,
  SUM(CASE WHEN LOWER(Tipo) REGEXP 'imposto sobre (taxas|comiss|assinatura)' THEN Débito ELSE 0 END) AS imp_comissoes
FROM transactions;

-- Último ciclo
WITH latest AS (
  SELECT "Ciclo Pagamento" FROM transactions
  WHERE "Data do ciclo de faturamento" IS NOT NULL
  ORDER BY "Data do ciclo de faturamento" DESC LIMIT 1
)
SELECT
  SUM(CASE WHEN LOWER(Tipo) REGEXP '(taxa|taxas|taxa do vendedor|taxa de assinatura)' THEN Débito ELSE 0 END) AS comissoes,
  SUM(CASE WHEN LOWER(Tipo) REGEXP 'imposto sobre (taxas|comiss|assinatura)' THEN Débito ELSE 0 END) AS imp_comissoes
FROM transactions
WHERE "Ciclo Pagamento" = (SELECT "Ciclo Pagamento" FROM latest);

5.4 Reembolsos (incluindo impostos) – acumulado e último ciclo:

-- Acumulado
SELECT
  SUM(CASE WHEN LOWER(Tipo) LIKE 'valor do pedido de reembolso%' THEN Crédito ELSE 0 END)
+ SUM(CASE WHEN LOWER(Tipo) LIKE 'imposto sobre o valor do pedido de reembolso%' THEN Crédito ELSE 0 END)
AS reembolsos_incl_imp
FROM transactions;

-- Último ciclo
WITH latest AS (
  SELECT "Ciclo Pagamento" FROM transactions
  WHERE "Data do ciclo de faturamento" IS NOT NULL
  ORDER BY "Data do ciclo de faturamento" DESC LIMIT 1
)
SELECT
  SUM(CASE WHEN LOWER(Tipo) LIKE 'valor do pedido de reembolso%' THEN Crédito ELSE 0 END)
+ SUM(CASE WHEN LOWER(Tipo) LIKE 'imposto sobre o valor do pedido de reembolso%' THEN Crédito ELSE 0 END)
AS reembolsos_incl_imp
FROM transactions
WHERE "Ciclo Pagamento" = (SELECT "Ciclo Pagamento" FROM latest);

5.5 Reserva presa (saldo) e ciclo da última constituiçăo (sem resumo oficial):

-- Saldo estimado via manuais + keywords
WITH res AS (
  SELECT SUM(real) AS saldo
  FROM transactions
  WHERE (LOWER(Tipo) LIKE '%fatura manual%' OR LOWER(Tipo) LIKE '%cr%dito manual%')
    AND LOWER(Descriçăo) REGEXP '(reserv|reten|hold|escrow)'
)
SELECT COALESCE(saldo,0) AS reserva_saldo FROM res;

-- Último ciclo com constituiçăo (valor negativo)
SELECT "Ciclo Pagamento"
FROM (
  SELECT "Ciclo Pagamento", SUM(real) AS v
  FROM transactions
  WHERE (LOWER(Tipo) LIKE '%fatura manual%' OR LOWER(Tipo) LIKE '%cr%dito manual%')
    AND LOWER(Descriçăo) REGEXP '(reserv|reten|hold|escrow)'
  GROUP BY 1
)
WHERE v < 0
ORDER BY "Ciclo Pagamento" DESC
LIMIT 1;

5.6 Conciliaçăo Net por ciclo vs TRF (+0/+7):

WITH cycle AS (
  SELECT "Ciclo Pagamento", MAX("Data do ciclo de faturamento") AS end_dt,
         SUM(Crédito) - SUM(Débito) AS net
  FROM transactions
  GROUP BY 1
),
trf AS (
  SELECT data, SUM(valor) AS v FROM bank_trf GROUP BY 1
)
SELECT
  "Ciclo Pagamento",
  CAST(end_dt AS DATE) AS cycle_end,
  net,
  (SELECT SUM(v) FROM trf WHERE data BETWEEN CAST(end_dt AS DATE) AND (CAST(end_dt AS DATE) + INTERVAL 7 DAY)) AS trf_0_7,
  ((SELECT SUM(v) FROM trf WHERE data BETWEEN CAST(end_dt AS DATE) AND (CAST(end_dt AS DATE) + INTERVAL 7 DAY)) - net) AS diff
FROM cycle
ORDER BY cycle_end;

6. ETL – Normalizaçăo e Ingestăo
Funçőes base para normalizar cabeçalhos, converter datas, numerificar e calcular coluna real (Crédito–Débito).

# etl_ingest.py (excerto)
REN = {
  "CrĂ©dito":"Crédito","DĂ©bito":"Débito","DescriĂ§ĂŁo":"Descriçăo",
  "NĂşmero da fatura":"Nş da fatura","NĂşmero da transaĂ§ĂŁo":"Nş da transaçăo",
  "RĂłtulo da categoria":"Rótulo da categoria"
}

def load_transactions(df):
    df = df.rename(columns={c: REN.get(c,c) for c in df.columns})
    for c in ["Crédito","Débito","Valor"]:
        if c in df: df[c] = pd.to_numeric(df[c], errors="coerce").fillna(0.0)
    for c in ["Data Criaçăo","Data Recebida","Data Transaçăo","Data do ciclo de faturamento"]:
        if c in df: df[c] = pd.to_datetime(df[c], errors="coerce", dayfirst=True)
    if "Crédito" in df and "Débito" in df:
        df["real"] = df["Crédito"] - df["Débito"]
    return df

7. API – Uploads e KPIs (FastAPI)
Endpoints mínimos:

POST /upload/transactions  -> recebe XLSX/CSV, normaliza e insere em transactions
POST /upload/trf           -> recebe XLSX/CSV, mapeia data/valor e insere em bank_trf
GET  /kpis/prazos          -> média/mín/máx (itens)
GET  /kpis/comissoes_acum  -> comissőes e imposto (acumulado)
GET  /kpis/comissoes_ult   -> comissőes e imposto (último ciclo)
GET  /kpis/reembolsos_acum -> reembolsos incl. imposto (acumulado)
GET  /kpis/reembolsos_ult  -> reembolsos incl. imposto (último ciclo)
GET  /kpis/reserva_saldo_est          -> saldo de reserva (estimado)
GET  /kpis/reserva_ult_ciclo_funding  -> último ciclo de constituiçăo
GET  /kpis/net_vs_trf      -> conciliaçăo por ciclo

Notas finais:

* - Para relatório de desvios contratuais, fornecer tabela de acordos (categoria/SKU/canal ? % comissăo).
* - Todos os cálculos usam sinal real (Crédito–Débito). Reconciliaçăo por ciclo deve dar diferença ? 0, garantindo consistęncia.
